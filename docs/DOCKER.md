# Docker

The repository includes a Dockerfile and Compose file for local evaluation.

## Quick Start

```bash
docker compose up --build
```

Then open:

```txt
http://localhost:8080/
https://localhost:8443/
```

The HTTPS endpoint uses the local self-signed certificate generated by `rpsiod`, so browsers and `curl` will warn unless you explicitly trust it.

## Mounted Paths

Compose mounts only the files you are expected to edit:

```txt
./config              -> /etc/rpsiod
./docker/www          -> /var/www/rpsiod-example
./www/rpsiod-errors   -> /var/www/rpsiod-errors
```

Container-managed data is stored in named volumes:

```txt
rpsiod-cache -> /var/cache/rpsiod
rpsiod-logs  -> /var/log/rpsiod
rpsiod-ssl   -> /var/lib/rpsiod
```

Runtime PID/socket files live in an in-memory tmpfs at `/run/rpsiod`.

## No Overwrite Behavior

The container entrypoint creates runtime directories and copies default config or website files only when the target file/directory is missing. It does not overwrite existing files in mounted config or website directories.

## Useful Commands

Build and run:

```bash
docker compose up --build
```

Run in the background:

```bash
docker compose up --build -d
```

View logs:

```bash
docker compose logs -f
```

Validate config in the container:

```bash
docker compose exec rpsiod rpsiod configtest
docker compose exec rpsiod rpsiod doctor
```

Stop:

```bash
docker compose down
```

Remove container-managed cache/log/SSL volumes:

```bash
docker compose down -v
```
